📋 Onboarding — Get Your Firebase Credentials

Follow these 6 steps to get every credential you need. After step 6, paste your config below and you are done.

  1. Step 1 — Create a Firebase project

    1. Open https://console.firebase.google.com and sign in.
    2. Click "Add project" → give it a name → continue.
    3. You can disable Google Analytics for now (not required for FCM).
    4. Wait for the project to be created, then click Continue.

  2. Step 2 — Register a Web App

    1. On the project dashboard, click the </> (Web) icon under "Get started by adding Firebase to your app".
    2. Give the app a nickname (e.g., "push demo") and click Register app.
    3. Do NOT check "Firebase Hosting" unless you plan to deploy there.
    4. You will see a code snippet with a firebaseConfig object. Copy this entire object — you'll paste it below in Step 7.
    // Example - yours will have different values:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abc123..."
};

  3. Step 3 — Get the VAPID (Web Push) Key

    This is the public key the browser uses to subscribe to push.

    1. In Firebase Console, click the ⚙️ gear icon → Project settings.
    2. Open the Cloud Messaging tab.
    3. Scroll to "Web configuration" → "Web Push certificates".
    4. Click "Generate key pair" if no key exists.
    5. Copy the key (it starts with B... and is ~88 chars). Paste it below in Step 7.

  4. Step 4 — Generate the Service Account JSON (for the server)

    This grants your backend permission to send pushes.

    1. In Project Settings, open the Service accounts tab.
    2. Click "Generate new private key" → confirm.
    3. A JSON file is downloaded. Rename it to service-account.json.
    4. Move it to the project folder: server/service-account.json
    5. ⚠️ Never commit this file to git. It is already in .gitignore.

  5. Step 5 — (Optional) Enable Cloud Messaging API (V1)

    On newer Firebase projects this is enabled by default. If sending fails with a 403 / API disabled error:

    1. Open Cloud Messaging API in Google Cloud Console.
    2. Make sure the right project is selected (top bar).
    3. Click Enable if it isn't already.

  6. Step 6 — Start the server

    In a terminal, from this project folder:

    npm install
npm start

    Then open http://localhost:3000 (this page). Browsers only allow notifications from https:// or localhost.

⚙️ Step 7 — Paste Your Firebase Web Config

Paste the values from Step 2 and Step 3. They are saved in your browser localStorage only — no upload.

🛠 Troubleshooting

Permission says "denied" — how do I reset it?

In Chrome: click the 🔒 lock icon in the address bar → Site settings → reset Notifications to "Ask".

"messaging/unsupported-browser" error

FCM requires HTTPS or localhost, plus support for Service Workers and the Push API. Safari requires version 16+ on iOS.

"Registration token is not valid"

Tokens can expire. Delete the token in your app and call getToken() again. Also confirm the VAPID key matches your Firebase project.

Sending fails with 403 / API disabled

Enable the Cloud Messaging API at this link for your project.

Service worker not found / 404 on firebase-messaging-sw.js

The service worker file must be served from the site root, with exactly that filename. Confirm it loads at http://localhost:3000/firebase-messaging-sw.js.
